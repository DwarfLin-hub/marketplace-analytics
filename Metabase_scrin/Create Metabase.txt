КОММИТ МЕТАБЕЙЗ РАЗВОРАЧИВАНИЕ ЛОКАЛЬНО И КАК ПРОЕКТ ЗАПУСКАЕТСЯ

МЕТАБЕЙЗ Локально на ноутбуке

Для этого мы скачиваем приложение Docker Desktop
устанавливаем его и запускаем
После запуска заходим в командный текст
В командной строке пишем: docker run -d -p 3000:3000 --name метабаза -e MB_DB_TYPE=postgres -e MB_DB_DBNAME=marketplace -e MB_DB_PORT=5432 -e MB_DB_USER=analyst -e MB_DB_PASS=ваш_пароль -e MB_DB_HOST=host.docker.internal метабаза/метабаза (пояснение: -d – запуск в фоновом режиме. -p 3000:3000 – проброс порта (локальный порт 3000 на порт 3000 контейнера). --name метабаза – имя контейнера. -e – переменные окружения для подключения к PostgreSQL: MB_DB_TYPE=postgres – тип БД. MB_DB_DBNAME=marketplace – имя базы. MB_DB_PORT=5432 – порт ПостгреSQL. MB_DB_USER=аналитик – пользователь БД. MB_DB_PASS=strong_password – пароль. MB_DB_HOST=host.docker.internal – специальный адрес для доступа из контейнера к вашему хосту (где работает PostgreSQL). )
После проверки работает ли докер
Пишем в командной строке docker ps – метабаза контейнера должна находиться в статусе Up.
Если все хорошо в откроем браузер и перейдем по адресу http://localhost:3000 . Увидим страницу настройки Метабазы.
Выполним первоначальную версию Метабазы

Создаем учётную запись администратора (имя, адрес электронной почты, пароль).
На шаге подключения к БД Metabase автоматически используем параметры из среды окружения – просто нажимаем «Сохранить».
Если подключение не удаётся, проверим:
Запущен ли PostgreSQL (служба postgresql-x64-16 в Services.msc).
В файле postgresql.conf должно быть Listen_addresses = '*'.
В pg_hba.conf добавлена ​​строка host all all 0.0.0.0/0 md5.
После изменений перезапущенным PostgreSQL.
Замените Host вместо host.docker.internal на ip, который можно увидеть, если в командной строке выбрать команду ipconfig.
Создаем Дашборды

В Метабазе создаем дашборд: «+ Новое» → «Панель управления»
В метабазе нажимаем «Задать вопрос» → «Нативный запрос» (SQL).
Вводим один из запросов из списка ниже (раздел «SQL-запросы для дашборда»).
Динамика выручки по дням (SQL: SELECT Purchase_date, SUM(total_price) AS выручка ОТ продаж GROUP BY Purchase_date ORDER BY Purchase_date; Предварительный просмотр: Линейный график (X – Purchase_date, Y – выручка). Название вопроса: «Выручка по дням» )
Нажимаем «Выполнить», убеждаемся, что данные указаны, затем «Сохранить».
Повторяем для всех нужных метрик. (Количество уникальных клиентов по дням SQL: SELECT Purchase_date, COUNT(DISTINCT client_id) AS daily_clients FROM sales GROUP BY Purchase_date ORDER BY Purchase_date; Предварительный просмотр: Линейный график (X – Purchase_date, Y – daily_clients).: «Активные клиенты по дням»
Топ-10 товаров по выручке SQL: SELECT product_id, SUM(total_price) AS доход ОТ продаж ГРУППИРОВАТЬ ПО Product_id ORDER BY доход DESC LIMIT 10; Предварительный просмотр: Столбчатая диаграмма (X – Product_id, Y – доход). Название: «Топ-10 товаров по выручке»

Топ-10 товаров по количеству продаж SQL: SELECT product_id, SUM(quantity) AS total_quantity FROM sales GROUP BY Product_id ORDER BY total_quantity DESC LIMIT 10; Предварительный просмотр: Столбчатая диаграмма (X – product_id, Y – total_quantity). Название: «Топ-10 товаров по количеству продаж»

Средний чек на клиента за последние 30 дней SQL: SELECT SUM(total_price) / COUNT(DISTINCT client_id) AS avg_check FROM sales WHERE Purchase_date >= CURRENT_DATE - INTERVAL '30 дней'; Предварительный просмотр: Скалярное значение (выберите «Число» в визуализациях). Название: «Средний чек на 30 дней»

Распределение покупок по часам SQL: SELECT (purchase_time_секунды / 3600) КАК час, COUNT(*) КАК заказов ОТ продаж ГРУППА ПО часам ЗАКАЗАТЬ ПО часам; Предварительный просмотр: Столбчатая диаграмма (X – час, Y – заказы). Название: «Активность по часам»

Соотношение полов клиентов SQL: SELECT пол, COUNT(DISTINCT client_id) AS cnt FROM клиентов GROUP BY пол; Предварительный просмотр: Круговая диаграмма. Название: «Половая структура клиентов»

Товары без продаж за последние 30 дней SQL: SELECT p.product_id FROM Products p LEFT JOIN sales ON p.product_id = s.product_id И s.purchase_date >= CURRENT_DATE - INTERVAL '30 дней' ГДЕ s.product_id IS NULL; Предварительный просмотр: Таблица (можно оставить по умолчанию). Название: «Товары без продаж (30 дней)» )

Затем создаем дашборд: «+ Новое» → «Панель управления», достраиваем все сохранённые вопросы.
Сохраним приборную панель в формате pdf.
